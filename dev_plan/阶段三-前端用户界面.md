# 阶段三：前端用户界面

## 目标

实现Flutter Web前端界面，提供用户友好的旅行规划体验。

## 主要任务

- 用户认证界面
- 行程列表页面
- 行程详情页面
- 地图集成界面
- 基础UI组件库

## 核心页面

- 登录/注册页面
- 行程列表页面
- 行程详情页面（地图+行程列表）
- API密钥配置页面
- 用户设置页面

## 界面设计要点

- 地图为主的交互界面
- 清晰的行程展示
- 支持PC横屏浏览
- 编辑模式和查看模式切换

## 技术要点

- Flutter Web
- 响应式设计
- 状态管理（Provider/Riverpod）
- 地图JavaScript桥接
- 认证使用SRP协议。
- API密钥加密，传输给服务端的应当是使用登陆密码加密过的密钥。
- 所有需要密钥的API访问都由前端发起，并将需要的信息传回后端。前端可以发送请求从后端获取加密过的密钥然后使用登陆密码解密。因此你应当保证相关的业务逻辑在前端实现。
